# Audio and Media

## Philosophy

O should not impose an unnecessary quality ceiling on audio and media workflows.

## Core goals

- high-quality local playback
- USB audio support
- USB MIDI support
- soundbank-backed music workflows
- O-MUS integration
- visible device routing and permissions

## Mother O role

Mother O provides:
- media controls
- audio player
- image viewer
- routing visibility
- device discovery

## Sub O role

Sub O may host:
- music composition workflows
- render/export jobs
- soundbank tools
- specialized media pipelines
