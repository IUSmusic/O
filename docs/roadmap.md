# Roadmap

## Phase 0 — architecture and tooling
- write master specs
- define Mother O services
- define Sub O manifest
- define O-IR, O-GFX, O-MUS v0.1
- establish repo structure

## Phase 1 — desktop-first simulation
- bring Mother O shell to Linux desktop
- implement Sub O lifecycle on desktop
- implement privacy ledger and local model manager
- build initial O UI shell and templates

## Phase 2 — phone proof of concept
- target Pixel 8-class device
- boot custom image
- validate kernel, storage, display, input, Wi-Fi, USB
- implement signed update and rollback
- run Mother O shell natively

## Phase 3 — Sub O developer environments
- containers first
- local-only AI mode
- brokered online AI mode
- dev template
- app studio template

## Phase 4 — media and expansion
- O-GFX pipeline
- O-MUS pipeline
- USB audio and MIDI support
- external AI accelerator support

## Phase 5 — production hardening
- trust store
- promotion flow
- safer update channel
- richer device policy
- additional supported devices

## Phase 6 — custom hardware
- choose SoC and modem strategy
- define physical ports and expansion model
- design around AI, audio, and docked workflows
