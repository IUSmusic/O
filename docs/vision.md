# Vision

## Product definition

O is a mobile phone and tablet operating system built for programmable, transparent, and safe computing.

More fully:

O is a full independent smartphone and tablet operating system built around a protected Mother O core, sandboxed Sub O worlds, and machine-native internal languages for software, graphics, and music.

## Why O exists

Modern operating systems optimize for fixed app packages, opaque system behavior, and limited user-directed evolution. O optimizes for:
- machine-native software understanding
- local-first AI assistance
- visible system activity
- safe experimentation in isolated worlds
- user-directed promotion of successful environments
- hardware adaptability and optional external expansion

## Intended experience

A user should be able to:
- create a calendar
- create a game
- create a temporary file converter
- build a custom workflow
- generate graphics and music assets
- use local or brokered AI to accelerate development

without turning the protected base system into a fragile experiment.

## What O is not

O is not:
- a launcher
- an Android or iOS skin
- a cloud shell
- a normal app-store-first system
- a system where the protected core edits itself freely
- a promise to support all devices at launch

## First audience

O is designed first for:
- programmers
- AI users
- technical builders
- creators comfortable with system-level tooling

## Foundational laws

1. Mother O is protected.
2. Sub O is the place for experimentation.
3. O is installable as the only OS on supported devices.
4. Local AI is the default.
5. Online AI is optional and brokered.
6. No Sub O can access Mother O without explicit gateways.
7. Promotion to Main O is staged and rollback-safe.
8. The system must remain visible, auditable, and recoverable.
