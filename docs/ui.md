# UI

## Mother O UI

Mother O UI should feel like a living system board.

### Core traits
- minimal
- calm
- text-first
- status-aware
- low animation
- high contrast
- fast

### Base layout
- top status strip
- optional pinned terminal
- editable 2×8 icon grid
- favorite Sub O list
- pinned app/tool area
- system messages area
- main O action button

### Interaction model
Every slot or module may support:
- move
- lock
- rename
- expand
- bind to action or world
- attach status badges

## Sub O template UI

Sub O should inherit the shell grammar but adapt to the use case.

### Dev template
- terminal-first
- file tree
- build/run status
- logs
- model panel

### App studio template
- preview
- code
- assets
- schema/data
- export

### Music template
- sequencer
- tracks
- soundbanks
- render/export

### Graphics template
- canvas/preview
- layers
- O-GFX panel
- export

### Research template
- sources
- notes
- citations
- agent panel
- output board
