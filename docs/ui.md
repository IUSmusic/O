# UI

## Mother O shell

Mother O should feel like a living system board.

### Core traits
- minimal
- calm
- text-first
- status-aware
- low animation
- high contrast
- fast

### Base layout
- top status strip
- optional pinned terminal
- editable 2×8 icon grid
- favorite Sub O list
- pinned app/tool area
- system messages area
- main O action button

### Slot behavior
Every slot may support:
- move
- lock
- rename
- expand
- bind to action, world, or app
- attach status badges
- act as template placeholder

## First-run behavior

The first-run flow should demonstrate template-based creation by building a browser from The Library with visible terminal progress, then pinning the resulting app into the shell.

## Relations view

Mother O includes a first-class Relations view showing the O Mapping graph:
- Sub O nodes
- trust edges
- firewall/network edges
- service and repo relationships
- warnings and broken relations

## Sub O templates

Sub O should reuse the shell grammar while specializing for context.

### Dev template
- terminal-first
- file tree
- build/run state
- logs
- model panel

### App studio template
- preview
- code
- assets
- schema/data
- export

### Graphics template
- preview/canvas
- layers
- O-GFX panel
- export

### Music template
- sequencer
- tracks
- soundbanks
- render/export

### Research template
- sources
- notes
- citations
- agent panel
- output board
