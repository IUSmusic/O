# Mother O

## Definition

Mother O is the protected, full-featured flagship environment of O.

It is:
- the safe boot environment
- the main dashboard
- the trust and policy root
- the broker and gateway authority
- the visible coordination layer for all Sub O worlds
- a complete operating system surface by default

## Service groups

### Boot, recovery, and integrity
- Boot Service
- Recovery Service
- Update Service
- Trust Service
- Superuser Authority Service

### Coordination and world management
- Sandbox Manager
- O Mapping Service

- Promotion Service

### Data and visibility
- Storage Authority Service

### AI and execution
- Local Model Runtime Service
- Online Model Broker Service
- Code Risk Service
- App Creation Service
- First-Run Bootstrap Service

### Network and device access
- Firewall Service
- Device Manager Service
- Device Gateway Service

### User-facing system services
- Shell Service
- App Registry and Runtime Service
- Editor and Studio Service
- Browser/Web Runtime Service
- Logs and Diagnostics Service
- Package / Module Manager Service
- Relations View Service

## App model in Mother O

Mother O runs:
- core system apps
- bootstrap-generated apps
- ordinary optional apps
- user-generated apps

Core system apps are not removable. Optional and generated apps are removable.

## First-run browser flow

A first-run template placeholder creates a lightweight browser/web runtime offline from The Library. The terminal shows progress. The resulting app is registered and pinned into the shell.
